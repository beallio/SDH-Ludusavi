Auto-Sync Status Icons

The icon + label that appear in the bottom status strip during auto-sync. Each state below renders with its real SVG and live animation.

Source of truth: src/surfaces/autoSyncStatusRenderer.tsx (icons & CSS), src/controllers/gameLifecycleController.tsx & syncthingMonitorMachine.ts (triggers). Generated reference — keep in sync if those change.

Blue — in progress / OK Amber — saved locally, attention Red — failure

Animated states (operation in progress)

Verifying Game Save
checking
Shown on game launch and game exit, before the auto-sync RPC check runs, while Ludusavi compares the local save against the backup. Only appears when the status bar is enabled for the tracked game.

Hide / persistence timing

  • Running states (checking, backing_up, restoring) — stay visible until the result arrives; safety ceiling auto-hides at RUNNING_STATUS_HIDE_CEILING_MS = 930 000 ms (~15.5 min). If they time out, the final result is suppressed.
  • Active Syncthing states (syncthing_pending_upload, syncthing_downloading, syncthing_uploading) — do not auto-hide; they stay up until a newer status replaces them.
  • All other result states — auto-hide after RESULT_HIDE_DELAY_MS = 2 000 ms.

Note: QAM action buttons (Back up / Restore now) show Decky's built-in <Spinner> while their operation runs — that is a separate component (src/components/qam/SpinnerButton.tsx), not part of this status-strip icon set.

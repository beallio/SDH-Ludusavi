Launch-Gated Backup Conflict Resolution

This chart documents the game-start path that safely hands a Steam bootstrap process into a verified frozen application scope, resolves Ludusavi backup recency, and thaws the scope in every outcome.

Startup Gate

App lifetime start

Steam reports a game start with app ID and process instance.

↓
Temporary bootstrap hold

Validate the PID owner and start-time identity, then hold only that bootstrap process during a short bounded handoff. This is not a successful gate by itself.

↓
Exact Steam app scope appears

Retry only the safe pre-scope app.slice state. Reject changed identity, malformed membership, and invalid paths immediately.

↓
Verified Steam app scope freeze

Run systemctl --user freeze with a sanitized child environment, verify requested and completed cgroup freezer state, release the bootstrap hold inside the frozen scope, and verify again. Processes that join later inherit the freeze.

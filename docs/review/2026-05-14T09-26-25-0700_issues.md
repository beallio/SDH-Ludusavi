## Issue 1

* **[Severity]** Medium
* **File & Function/Line:** `py_modules/sdh_ludusavi/service.py`, `SDHLudusaviService._match_game`, lines 713-721
* **Description:** The fuzzy substring guard uses `or`, so a one-character game name like `A` can match any longer title containing `a`. The comment says this is meant to avoid matching short names, but the condition allows it whenever the target title is longer than four characters. This can trigger backup or restore for the wrong game on launch/exit.
* **Vulnerable Code:**

```python
if normalized_input in normalized_target or normalized_target in normalized_input:
    # Minimum length check to avoid matching e.g. "A" to every game with "A"
    if len(normalized_input) > 4 or len(normalized_target) > 4:
        self.log("info", f"Matched '{game_name}' via fuzzy substring to '{game.name}'")
        return game
```

* **Proposed Fix:**

```python
if (
    len(normalized_input) > 4
    and len(normalized_target) > 4
    and (normalized_input in normalized_target or normalized_target in normalized_input)
):
    self.log("info", f"Matched '{game_name}' via fuzzy substring to '{game.name}'")
    return game
```

## Issue 3

* **[Severity]** Medium
* **File & Function/Line:** `py_modules/sdh_ludusavi/service.py`, `SDHLudusaviService._refresh_statuses_unlocked`, lines 633-643
* **Description:** Invalid adapter entries are caught, but the error path assumes `raw_game` has `.get()`. If Ludusavi or the adapter returns a non-mapping item, the exception handler raises another exception and aborts the entire refresh instead of skipping the bad item.
* **Vulnerable Code:**

```python
for raw_game in raw_statuses:
    try:
        game = self._coerce_game_status(raw_game)
        games.append(game)
    except Exception as exc:
        self.log(
            "error",
            f"Failed to parse status for game {raw_game.get('name')}: {exc}",
            "refresh",
        )
```

* **Proposed Fix:**

```python
from collections.abc import Mapping

for raw_game in raw_statuses:
    try:
        if not isinstance(raw_game, Mapping):
            raise TypeError(f"status entry must be a mapping, got {type(raw_game).__name__}")
        game = self._coerce_game_status(dict(raw_game))
        games.append(game)
    except Exception as exc:
        raw_name = raw_game.get("name") if isinstance(raw_game, Mapping) else "<unknown>"
        self.log(
            "error",
            f"Failed to parse status for game {raw_name}: {exc}",
            "refresh",
        )
```

## Issue 4

* **[Severity]** High
* **File & Function/Line:** `py_modules/pyludusavi/main.py`, `Ludusavi.backup` and `Ludusavi.restore`, lines 180, 255, 275, 331
* **Description:** Backup and restore default to `timeout=None`, so a hung Ludusavi process can hold the plugin operation lock indefinitely. In Decky this leaves sync operations permanently skipped as `operation_running` until process restart.
* **Vulnerable Code:**

```python
timeout: Optional[float] = None,  # Operations default to no timeout
...
response = self.executor.execute(args, mode="JSON", timeout=timeout)
```

* **Proposed Fix:**

```python
DEFAULT_OPERATION_TIMEOUT_SECONDS = 60

def backup(
    self,
    games: Optional[List[str]] = None,
    preview: bool = False,
    ...
    timeout: Optional[float] = DEFAULT_OPERATION_TIMEOUT_SECONDS,
) -> LudusaviResponse[LudusaviApiOutput]:
    ...
    response = self.executor.execute(args, mode="JSON", timeout=timeout)
    assert response is not None
    return response
```

Apply the same default to `restore`, or expose a service-level timeout that always passes a finite value.

## Issue 5

* **[Severity]** High
* **File & Function/Line:** `py_modules/sdh_ludusavi/service.py`, `SDHLudusaviService.__init__`, lines 188-206
* **Description:** The service logs selected environment variables and every variable starting with `DECKY_` or `FLATPAK_` into the plugin log buffer. This can disclose usernames, home paths, runtime paths, and any future token-like Decky/Flatpak variable to the frontend log viewer. Debug logs should use an allowlist of non-secret keys and redact path-heavy values unless explicitly requested.
* **Vulnerable Code:**

```python
_filtered_env = {
    k: v
    for k, v in os.environ.items()
    if k in _relevant_keys or k.startswith(("DECKY_", "FLATPAK_"))
}
self.log("debug", f"Filtered environment variables: {_filtered_env}", "init")
```

* **Proposed Fix:**

```python
_allowed_env_keys = {
    "LANG",
    "DECKY_VERSION",
    "DECKY_PLUGIN_RUNTIME_DIR",
    "DECKY_PLUGIN_SETTINGS_DIR",
    "FLATPAK_ID",
}
_filtered_env = {
    key: ("<set>" if "DIR" in key or key.endswith("HOME") else value)
    for key, value in os.environ.items()
    if key in _allowed_env_keys
}
self.log("debug", f"Environment summary: {_filtered_env}", "init")
```

## Issue 6

* **[Severity]** Medium
* **File & Function/Line:** `py_modules/sdh_ludusavi/ludusavi.py`, module import, line 9
* **Description:** Importing this module globally clears `LD_LIBRARY_PATH` for the entire Python process. That side effect affects every later subprocess and imported native extension in the Decky backend, not just Ludusavi. Environment changes should be scoped to the Ludusavi subprocess.
* **Vulnerable Code:**

```python
FLATPAK_ID = "com.github.mtkennerly.ludusavi"
os.environ["LD_LIBRARY_PATH"] = ""
```

* **Proposed Fix:**

```python
FLATPAK_ID = "com.github.mtkennerly.ludusavi"

def _ludusavi_env() -> dict[str, str]:
    env = os.environ.copy()
    env["LD_LIBRARY_PATH"] = ""
    return env
```

Then pass `_ludusavi_env()` into the executor paths that invoke Ludusavi, instead of mutating `os.environ` at import time.

## Issue 7

* **[Severity]** Medium
* **File & Function/Line:** `main.py`, `_run_blocking`, lines 230-256
* **Description:** Async cancellation does not stop the background thread. `_call` re-raises `asyncio.CancelledError`, but `_run_blocking` has already started a daemon thread that can continue running a backup/restore and holding the service lock after the caller is gone. This creates confusing state and can keep later operations blocked.
* **Vulnerable Code:**

```python
thread = threading.Thread(target=runner, name="sdh-ludusavi-worker", daemon=True)
thread.start()
while not complete.is_set():
    await asyncio.sleep(0.01)
thread.join()
if error is not None:
    raise error
return value
```

* **Proposed Fix:**

```python
async def _run_blocking(callback: Any) -> Any:
    loop = asyncio.get_running_loop()
    task = loop.run_in_executor(None, contextvars.copy_context().run, callback)
    try:
        return await task
    except asyncio.CancelledError:
        decky.logger.warning("SDH-ludusavi operation was cancelled while worker may still be running")
        raise
```

For a complete fix, pair this with finite Ludusavi subprocess timeouts so cancelled operations cannot run forever.

## Issue 9

* **[Severity]** Medium
* **File & Function/Line:** `src/types/steam-globals.d.ts`, global declarations, lines 4-9; `package.json`, dependencies
* **Description:** TypeScript validation currently fails. The local globals redeclare `SteamClient` and `appStore` in a way that conflicts with `@decky/ui`'s bundled declarations, and `@decky/api` references `react-router` without a local dependency. Because Rollup still builds, this can let frontend type regressions ship unnoticed.
* **Vulnerable Code:**

```typescript
declare global {
  const SteamClient: SteamClientGlobal | undefined;
  interface Window {
    SteamClient?: SteamClientGlobal;
    appStore?: AppStoreGlobal;
  }
  var appStore: AppStoreGlobal | undefined;
}
```

* **Proposed Fix:**

```typescript
declare global {
  interface Window {
    SteamClient?: SteamClientGlobal;
    appStore?: AppStoreGlobal;
  }
}
```

Then consume the existing Decky globals through runtime guards:

```typescript
function getSteamClient(): SteamClientGlobal {
  const client = (globalThis as { SteamClient?: unknown }).SteamClient ?? window.SteamClient;
  if (!client || typeof client !== "object" || !("Apps" in client)) {
    throw new Error("SteamClient.Apps is unavailable in this frontend context.");
  }
  return client as SteamClientGlobal;
}
```

Add the missing type/runtime dependency if required by the Decky package version:

```json
{
  "dependencies": {
    "react-router": "^6.30.0"
  }
}
```

## Issue 10

* **[Severity]** Best Practice
* **File & Function/Line:** `py_modules/sdh_ludusavi/ludusavi.py`, `PyludusaviAdapter.get_aliases`, lines 71-80
* **Description:** Alias discovery swallows all exceptions. If config parsing, permissions, or Ludusavi invocation fails, matching silently degrades and users only see unexplained missed auto-sync behavior. The service already has a log channel, so this should be visible at warning/debug level.
* **Vulnerable Code:**

```python
try:
    config = self._client.config_show().data
    for game in config.get("customGames", []):
        name = game.get("name")
        alias = game.get("alias")
        if name and alias:
            aliases[name] = alias
except Exception:
    pass
return aliases
```

* **Proposed Fix:**

```python
try:
    config = self._client.config_show().data
except Exception as exc:
    LOGGER.warning("Unable to load Ludusavi aliases: %s", exc)
    return aliases

for game in config.get("customGames", []):
    if not isinstance(game, dict):
        continue
    name = game.get("name")
    alias = game.get("alias")
    if name and alias:
        aliases[str(name)] = str(alias)
return aliases
```

# v0.4.2 — Security bump and Deck screenshot tooling

**No plugin runtime changes in this release.** Nothing under `py_modules/` or `src/` changed since v0.4.1, so the panel, QAM, and backup/restore behavior are identical. This is a maintenance release covering the build toolchain and developer tooling — if you are on v0.4.1 there is no functional reason to update.

## Security

- Bumped the pinned `brace-expansion` overrides to `1.1.16` / `2.1.2` to clear **GHSA-3jxr-9vmj-r5cp** (high-severity ReDoS via exponential expansion of consecutive non-expanding `{}` groups). The package is a transitive dev-only dependency reached through `@decky/rollup` → `glob`/`rimraf` → `minimatch`; it is not shipped in the plugin ZIP, so the exposure was to the build environment, not to installed plugins. The previous pins (`1.1.13` / `2.0.3`) predated the advisory. (#6f39985)

## Developer Tooling

- Added `scripts/deck_screenshot.sh` for capturing the Deck's current screen over SSH during UI work. The Deck runs a gamescope session where `grim` and `spectacle` cannot attach, so the script drives `gamescopectl screenshot`, sets `XDG_RUNTIME_DIR` for the graphical session, polls for a stable file size (gamescopectl returns before the PNG is written), copies the result back, and cleans up the remote temp file. Documented under "Deck Screenshots" in `DEVELOPMENT.md`. (#ed67ba5)

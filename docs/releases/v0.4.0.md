# v0.4.0 — Per-game sync controls

Minor release adding per-game sync opt-outs, a disabled-state notice on game start and exit, and fixes to which game the panel displays.

## Added

- **Per-game sync toggle.** Automatic backup on game start and exit can now be turned off for individual games from the QAM panel, instead of being all-or-nothing. The preference is stored per game, exposed through the backend API, and applied by the lifecycle gate, so a disabled game skips automatic sync entirely and bypasses the launch gates rather than waiting on them. Toggling is optimistic in the UI and reconciles against the persisted result. (f0f78a2 … 81e1049)
- **Disabled notice on game start and exit.** Launching or quitting a game with sync disabled now shows a status strip confirming no backup ran, rather than silently doing nothing. The notice was initially start-only; it appears on both paths. It uses the Lucide `save-off` glyph, which reads as "saving is off" rather than as a generic prohibition sign. (5be3e71, 003bf25, 2535892)

## Fixed

- **The panel no longer loses or misreports the displayed game.** Displayed-game state was coupled to the settings store, so a refresh could clear a valid live selection and hydration could leave the panel showing nothing. Display state is now tracked separately, seeded during hydration, and preserves a valid selection across refresh. The invariant is documented in `DEVELOPMENT.md`. (5841ed6, c859c60, b715b6f)

## Build

- **Dev-release builds survive a failed publish.** A GitHub Releases outage failed the publish step three times in a row after the build had already passed quality gates, packaging, and ZIP validation — and because the packaged output was never uploaded, each validated build was discarded with the run, forcing a tag deletion and full rebuild. `dev-release.yml` now uploads `out/` before publishing and with `if: always()`, so a retry can be served from a build that already passed every gate. (6b88b02)
